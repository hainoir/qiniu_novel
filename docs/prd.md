# AI 小说转剧本工具 PRD

## 1. 产品定位

本项目把小说内容编译为 AI 视频生成模型可理解的结构化创作资产，使小说作者能够从文本故事快速进入 AI 漫剧生产流程。

项目不是单纯的小说摘要工具，也不是直接生成视频的模型，而是位于小说文本和 AI 视频模型之间的结构化中间层：

```text
小说章节
  -> 剧本 YAML
  -> 分镜 YAML
  -> video_generation_jobs
  -> AI 视频模型生成单元视频
  -> 漫剧剪辑与发布
```

当前 MVP 已完成第一步：把 3 个章节以上的小说文本转换为结构化剧本 YAML。后续版本会继续把剧本拆成分镜，并把每个分镜转成可投递给视频生成模型的单元视频任务。

## 2. 为什么要做

AI 视频模型已经能够根据 prompt 生成短视频片段，但小说作者很难直接把长篇小说稳定地转成视频模型可执行的 prompt。

小说文本通常包含：

- 长篇叙事和跨章节线索。
- 心理描写和隐含动机。
- 场景跳转、时间跳转和视角切换。
- 不直接对应镜头语言的文学表达。

而视频生成模型通常需要：

- 明确的主体、动作、地点和时间。
- 明确的镜头大小、机位、镜头运动和构图。
- 明确的时长、画幅、视觉风格和负向提示。
- 可拆分、可排队、可重试的短片段任务。

因此，本项目要解决的是“小说到视频生成任务”之间的结构化断层，让作者不用从零学习剧本、分镜和视频模型 prompt，也能把故事逐步拆成 AI 漫剧生产资产。

## 3. 目标用户

- 小说作者：希望把已有小说快速改编为剧本、分镜或漫剧短视频。
- AI 漫剧创作者：需要把文本故事拆成可批量生成的视频片段。
- 内容团队：需要把小说 IP 快速可视化，验证角色、场景和剧情节奏。
- AI 视频工具开发者：需要一层可解析、可校验的故事中间表示。

## 4. 核心场景

1. 作者粘贴至少 3 个小说章节。
2. 系统生成结构化剧本 YAML，保留来源章节、角色、场景、动作和对白。
3. 作者编辑剧本 YAML，修正角色、节奏和细节。
4. 系统将剧本场景拆成分镜 YAML，每个镜头包含时长、景别、机位、动作和视觉提示。
5. 系统将分镜转成 `video_generation_jobs`，每个任务包含 prompt、时长、画幅、负向提示和参考素材字段。
6. 后续版本可将任务投递给 Sora、Veo、Runway、可灵等视频生成模型。

## 5. MVP 范围

当前 MVP：

- 输入 3 个章节以上小说文本。
- 输出结构化剧本 YAML。
- 支持复制、下载和继续编辑 YAML。
- 提供剧本 YAML Schema 文档。
- 校验 AI 输出是否为合法 YAML 和可用剧本结构。

当前 MVP 不包含：

- 真实视频生成 API 调用。
- 角色形象图生成。
- 视频剪辑和音频合成。
- 多人协作和版本管理。

## 6. 后续版本

V2：剧本 YAML -> 分镜 YAML

- 将剧本场景拆分为多个镜头。
- 为每个镜头生成时长、景别、机位、镜头运动和视觉 prompt。
- 保留分镜到剧本场景的追溯关系。

V3：分镜 YAML -> video_generation_jobs

- 将每个镜头转成视频生成任务。
- 输出 provider-agnostic 的通用任务结构。
- 支持后续映射到不同视频模型 API。

V4：视频模型适配器

- 增加 Sora、Veo、Runway、可灵等 provider adapter。
- 将通用任务转换为具体 API 请求。
- 支持任务状态追踪、失败重试和结果下载。

## 7. 成功标准

- README 能清楚说明项目服务 AI 漫剧生产，而不是普通文本改写。
- 剧本、分镜和视频任务都有明确 Schema。
- 每一层输出都可以被人类编辑，也可以被程序校验。
- 每个分镜和视频任务都能追溯到来源剧本场景。
- 视频任务字段能覆盖主流视频模型 API 的核心输入：prompt、duration、aspect_ratio、negative_prompt、reference_assets、seed 和 status。
